# Project Sketch — CodeTrip 기획 초안

## 1. 프로젝트 주제 선정 배경

- 벤치마킹 레퍼런스: [한국관광공사 대한민국 구석구석](https://korean.visitkorea.or.kr/main/main.do)
- 최종 선정 주제: **여행지 소개 + 날씨 기반 랜덤 추천**

---

## 2. 프로젝트 최종 방향

### 컨셉: "코드를 짜듯 꼼꼼하게 설계된 여행 가이드"

> **CodeTrip** — 코드 주석처럼 세밀한 여행 가이드를 제공하는 서비스

- 여행지 상세 정보를 코드 주석(`// TIP:`, `/* 주의사항 */`) 스타일의 UI로 표현
- 날씨 API를 연동하여 현재 날씨에 맞는 여행지를 추천하는 기능

### 페르소나

| 항목 | 내용 |
|------|------|
| 이름 | 정해진 (26세, 취준생 / 주니어 개발자) |
| 특징 | 어디 갈지 고르는 선택 장애, 날씨 좋으면 일단 나가고 싶은 타입 |
| 니즈 | 지금 내 상황(날씨/위치/시간)에 맞는 딱 한 곳 추천받기 |

---

## 3. 핵심 기능 아이디어

### 3.1 지역별/테마별 여행지 소개
- 여행지 상세 페이지 (게시판 + 댓글 형식)
- 설명은 코드 주석 스타일 UI로 표현
- Kakao Maps API로 위치 마커 표시
- 지역별(제주/부산/서울), 카테고리별(맛집/명소/숙소) 다중 필터

### 3.2 나의 여행 리스트 (즐겨찾기)
- LocalStorage로 새로고침 후에도 목록 유지
- 장바구니 형식의 UI

### 3.3 날씨 기반 여행지 랜덤 추첨
- 날씨 API로 현재 기상 상태 파악
- 날씨에 따른 필터링 로직:
  - 맑음 ☀️ → 야외 명소 / 인생샷 명소
  - 비/눈 🌧️ → 실내 전시회 / 문화 공간
  - 구름 ☁️ → 산책로 / 로컬 행사
- 슬롯머신 스타일 랜덤 뽑기 UI (Framer Motion 활용 예정)

### 3.4 축제/행사 정보 (보류)
- 기간별 지역 축제 안내 + 지도

---

## 4. 프론트엔드 구현 포인트

- **날씨 테마 동적 UI**: 날씨 상태에 따라 배경색/아이콘 변경
- **Skeleton UI**: API 로딩 중 스켈레톤 화면
- **Debounce**: 검색 성능 최적화
- **Lazy Loading**: 이미지 초기 로딩 최적화
- **반응형**: 모바일 퍼스트 디자인
- **카카오맵 로드뷰**: 뽑힌 장소 사전 확인

---

## 5. 기술 스택 후보 (선정 당시 검토)

| 구분 | 도구 | 용도 |
|------|------|------|
| 데이터베이스 | Supabase / MySQL | DB + 이미지 업로드 |
| 데이터 페칭 | TanStack Query | API 호출 관리 |
| 디자인 | Tailwind CSS | UI 퀄리티 |
| 배포 | Vercel / Netlify | 정적 배포 |

**최종 선택**: React (Vite) + Tailwind CSS + MySQL (EC2) + 공공데이터 API
