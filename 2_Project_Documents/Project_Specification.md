## 2026.04.23
1. **축제 및 행사 정보 시스템 고도화**
    - **서버 사이드 정렬**: 사용자가 '날짜순(오름차순/내림차순)' 선택 시 서버 캐시 데이터를 기준으로 즉각적인 정렬 결과를 반환.
    - **페이지네이션**: 대량의 축제 데이터를 8개 단위로 서버에서 분할 응답하여 클라이언트 부하 감소.
2. **API 안정성 및 에러 관리**
    - **Proxy 회로 차단기**: 공공 데이터 API의 429(Too Many Requests) 에러 발생 시 자동 차단 및 회복 메커니즘 가동.
    - **프록시 캐싱**: 동일 상세 정보 요청에 대해 서버 메모리 캐시를 활용하여 외부 API 호출 횟수 90% 이상 절감.
3. **위시리스트 타임존 보정 시스템**
    - **순수 날짜 데이터 처리**: 타임존 정보가 배제된 YYYY-MM-DD 문자열 기반의 데이터 파이프라인 구축.
    - **로컬 파싱 알고리즘**: 브라우저 환경에 관계없이 입력한 날짜를 정확하게 표시하는 `parseLocalDate` 유틸리티 도입.

## 2026.04.27
1. **탐색(Explore) 및 정렬(Sort) 기능 통합**
    - **서버 사이드 캐시 기반 탐색**: `feature/explore_sort` 브랜치의 핵심 로직을 병합하여 여행지 및 축제 정보에 대한 다각도 정렬 기능(생성일순, 수정일순, 날짜순)을 공식 반영.
    - **캐시 초기화 전략**: 서버 구동 시 약 6만 건의 여행 데이터를 로딩하고, 축제 데이터 2천 건을 별도 정규화하여 메모리에 상주시켜 검색 속도 극대화.
2. **위시리스트 폴더 관리 정밀화**
    - **날짜 정합성 보장**: 폴더의 시작/종료 날짜 수정 시 발생하는 '날짜 밀림 오류'를 서버-클라이언트 전 구간 문자열 데이터 바인딩을 통해 해결.
    - **메타데이터 동기화**: MySQL `dateStrings: true` 설정을 통해 데이터베이스 레벨부터 타임존 오프셋 개입 원천 차단.
3. **사용자 경험(UX) 개선**
    - **부드러운 스크롤**: 페이지 이동 및 정렬 변경 시 화면 상단으로 부드럽게 이동하는 `window.scrollTo` 로직 통합.
    - **안전한 상태 업데이트**: `isMounted` 가드를 통해 컴포넌트 언마운트 후의 상태 업데이트를 방지하여 메모리 누수 및 에러 차단.
