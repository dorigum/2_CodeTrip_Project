# 프로젝트 명세서 (Project_Specification)

> 프로젝트 기획, 기술 스택, DB 설계 및 API 명세를 정의한 문서입니다.

## 1. 프로젝트 개요: CodeTrip
- **프로젝트명**: CodeTrip (Vibe Board + Tour Info)
- **목적**: 마크다운 기반의 여행 기록 공유 플랫폼과 공공데이터 기반 여행 정보 서비스 통합.
- **주요 기능**:
    - 위시리스트 폴더별 관리 시스템 및 일정 기반 여행 플래닝.
    - 서버 사이드 캐싱을 통한 6만 건 이상의 전국 여행지 초고속 검색 및 탐색.
    - 마크다운 게시판 시스템을 통한 고도화된 여행지 리뷰 및 정보 공유.
    - 위시리스트 폴더별 메모/체크리스트 기능을 통한 상세 여행 준비 지원.

---

## 2. 기술 스택 및 인프라 (Tech Stack)
### 2.1 Backend
- **Node.js (Express)**: RESTful API 서버 구축.
- **MySQL (v8.0)**: 관계형 데이터베이스 관리.
- **JWT (JSON Web Token)**: 사용자 인증 및 세션 관리 (유효기간 24시간).
- **Multer**: 프로필 및 게시판 이미지 업로드 관리 (용량 제한 5MB).
- **Axios**: 공공데이터 API 연동 및 프록시 서버 구현.

### 2.2 Frontend
- **React (Vite)**: 고성능 프론트엔드 환경 구축.
- **Zustand**: 전역 상태 관리 (Auth, Wishlist, Board, Explore 등).
- **Tailwind CSS**: 유틸리티 중심의 반응형 디자인 시스템.
- **React Router v7**: 복합 라우팅 및 동적 경로 관리.
- **React-Markdown**: 게시판 내 마크다운 렌더링 및 GFM 지원.

---

## 3. 데이터베이스 설계 (Database Schema)
- **users**: 사용자 기본 정보 및 프로필 이미지 경로.
- **wishlist_folders**: 사용자별 위시리스트 폴더 (이름, 일정 시작/종료일).
- **wishlists**: 폴더에 속한 개별 여행지 정보 (content_id, title, image_url 등).
- **wishlist_notes**: 폴더별 메모 및 체크리스트 (타입별 구분: LIST/MEMO).
- **travel_comments**: 여행지별 리뷰 및 좋아요 정보.
- **board_posts**: 마크다운 기반 게시글 및 태그, 조회수 관리.
- **board_comments**: 게시글 내 댓글 및 대댓글 구조 지원.

## 4. 핵심 기능 명세 (2026.04.27 업데이트)

### 4.1 탐색(Explore) 및 정렬 시스템 통합
- **서버 사이드 캐시**: 서버 구동 시 공공데이터 6만 건을 메모리에 적재하여 ms 단위의 검색 속도 보장.
- **다각도 정렬**: 생성일순, 수정일순, 행사 시작일순 등 다양한 정렬 기준 지원 (서버사이드 정렬).

### 4.2 위시리스트 타임존 보정 시스템
- **데이터 정합성**: 로컬 시간대와 UTC 간의 차이로 인해 날짜가 하루씩 밀리던 오류를 '순수 문자열(YYYY-MM-DD)' 기반 파이프라인으로 해결.
- **통합 관리**: 폴더별 일정 설정, 메모 작성, 여행지 추가/삭제 전 과정에서 일관된 사용자 경험 제공.

### 4.3 API 안정성 (Proxy & Circuit Breaker)
- **429 Error 방지**: 공공데이터 API의 호출 제한을 극복하기 위해 서버 프록시 캐시(2시간) 및 에러 발생 시 자동 차단 기능을 가동 중.
